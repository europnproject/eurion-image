# Eurion OS – Security Policy

## Responsible Disclosure
Se trovi una vulnerabilità:
- NON pubblicare dettagli in issue pubbliche
- apri una issue con label `security` senza dettagli tecnici sensibili
- oppure invia dettagli tramite canale privato (da definire)

## Obiettivi
- patch rapide
- advisory pubblici
- supply chain trasparente (SBOM + signing in roadmap)

## Scope iniziale
- configurazioni e build Eurion (eurion-image)
- Eurion Hub / Control Center (quando pubblicati)
